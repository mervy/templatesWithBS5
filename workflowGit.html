Fluxo de trabalho com Git e GitHub

Usando as branchs main, develop, feature, release e hotfix, onde os recursos são desenvolvidos nas branches de feature, unidos e testados na branch de release e depois implantados em produção na branch de main.

Criar um novo repositório no GitHub e cloná-lo para sua máquina local:

  • git clone https://github.com/seu-usuario/seu-repositorio.git
  • cd seu-repositorio

Criar as branchs principais:

  • git checkout -b main
  • git push -u origin main
  • git checkout -b develop
  • git push -u origin develop

Criar uma nova branch de feature para desenvolver um recurso:

  • git checkout -b feature/nome-do-recurso

Desenvolver o recurso na branch de feature e fazer commits regulares:

  • git add .
  • git commit -m "descrição do commit"

Quando o recurso estiver pronto para ser testado, mesclar a branch de feature na branch de release:

  • git checkout release
  • git merge --no-ff feature/nome-do-recurso
  • git push origin release

Testar a branch de release e fazer ajustes se necessário. Depois, mesclar a branch de release na branch de main:

  • git checkout main
  • git merge --no-ff release
  • git push origin main

Mesclar a branch de release na branch de develop para que os recursos desenvolvidos possam ser integrados em um próximo release:

  • git checkout develop
  • git merge --no-ff release
  • git push origin develop

Se houver um problema crítico na produção, criar uma branch de hotfix para corrigir o problema:

  • git checkout -b hotfix/nome-do-problema

Corrigir o problema na branch de hotfix e fazer commits regulares:

  • git add .
  • git commit -m "descrição do commit"

  • Mesclar a branch de hotfix na branch de main para corrigir o problema na produção:

  • git checkout main
  • git merge --no-ff hotfix/nome-do-problema
  • git push origin main

Mesclar a branch de hotfix na branch de develop para que a correção possa ser integrada em um próximo release:

  • git checkout develop
  • git merge --no-ff hotfix/nome-do-problema
  • git push origin develop

Esse é um exemplo de fluxo de trabalho possível com Git e GitHub usando as branchs main, develop, feature, release e hotfix. Você pode personalizar esse fluxo de acordo com as necessidades da sua equipe e do seu projeto.
